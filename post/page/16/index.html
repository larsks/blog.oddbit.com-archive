  5. Puppet, scope, and inheritance

    Published Tue, Aug 16, 2011

    I note this here because it wasn’t apparent to me from the Puppet documentation. If you have a Puppet class like this: class foo { File { ensure => file, mode => 600, } } And you use it like this: class bar { include foo file { '/tmp/myfile': } } Then /tmp/myfile will not be created. But if instead you do this: class bar inherits foo { file { '/tmp/myfile': } } It will be created with mode 0600. (more)
